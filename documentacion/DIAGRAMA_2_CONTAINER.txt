================================================================================
FLUJO DE DATOS:
================================================================================

FLUJO A: Cliente envía feedback
  Cliente → Twilio → Backend API → MongoDB
                  ↓
                OpenAI API

  1. Cliente envía WhatsApp
  2. Twilio recibe y envía webhook
  3. Backend recibe POST
  4. Backend guarda en MongoDB
  5. Backend llama OpenAI
  6. OpenAI retorna análisis
  7. Backend actualiza documento en MongoDB


FLUJO B: Gerente visualiza dashboard
  Frontend → Backend API → MongoDB → Frontend → Gerente

  1. Frontend hace polling (cada 10 seg)
  2. GET /api/sentimientos, /api/temas, etc.
  3. Backend consulta MongoDB
  4. MongoDB retorna datos
  5. Backend formatea JSON
  6. Frontend recibe y renderiza
  7. Gerente ve dashboard actualizado


================================================================================
PROTOCOLOS DE COMUNICACIÓN:
================================================================================

Twilio → Backend API
  • Protocolo: HTTPS
  • Método: POST
  • Endpoint: /webhook/whatsapp
  • Formato: application/x-www-form-urlencoded
  • Datos: Body, From, MessageSid, etc.

Backend API → OpenAI
  • Protocolo: HTTPS
  • Método: POST
  • Endpoint: /v1/chat/completions
  • Formato: JSON
  • Autenticación: Bearer token

Backend API → MongoDB
  • Protocolo: MongoDB Wire Protocol (sobre TCP)
  • Driver: PyMongo
  • Conexión: mongodb+srv:// (TLS)
  • Operaciones: insert_one, update_one, aggregate

Frontend → Backend API
  • Protocolo: HTTP/HTTPS
  • Método: GET
  • Formato: JSON
  • CORS: Habilitado
  • Polling: Cada 10 segundos
