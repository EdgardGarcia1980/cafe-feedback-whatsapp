SECUENCIA: Recibir y Procesar Mensaje de WhatsApp

1. [Twilio] → POST /webhook/whatsapp
   Body="El café estaba delicioso", From="whatsapp:+50312345678"

2. [Webhook Controller] → request.form.get('Body', 'From')
   Extrae: texto="El café estaba delicioso", numero="whatsapp:+50312345678"

3. [Webhook Controller] → db_service.save_message(texto, numero)

4. [Database Service] → MongoDB.insert_one({
     texto_mensaje: "El café estaba delicioso",
     numero_remitente: "whatsapp:+50312345678",
     timestamp: ISODate("2024-03-18T..."),
     sentimiento: null,
     tema: null,
     resumen: null
   })

5. [MongoDB] → Retorna inserted_id = "65f8a..."

6. [Database Service] → Retorna message_id = "65f8a..."

7. [Webhook Controller] → ai_service.analizar_sentimiento(texto)

8. [AI Service] → Construye prompt Few-shot + texto

9. [AI Service] → OpenAI.chat.completions.create(
     model="gpt-3.5-turbo",
     messages=[...],
     temperature=0.3
   })

10. [OpenAI API] → Retorna {
      choices: [{
        message: {
          content: '{"sentimiento":"positivo","tema":"Calidad del Producto",...}'
        }
      }]
    }

11. [AI Service] → JSON.parse(content)
    Valida campos, normaliza valores

12. [AI Service] → Retorna {
      sentimiento: "positivo",
      tema: "Calidad del Producto",
      resumen: "Cliente satisfecho con calidad del café",
      metadatos: {...}
    }

13. [Webhook Controller] → db_service.update_message_analysis(
      message_id, "positivo", "Calidad del Producto", "Cliente satisfecho...", {...}
    )

14. [Database Service] → MongoDB.update_one(
      {_id: ObjectId("65f8a...")},
      {$set: {sentimiento: "positivo", tema: "Calidad del Producto", ...}}
    )

15. [MongoDB] → Confirma actualización

16. [Webhook Controller] → return '', 200

17. [Twilio] → Recibe 200 OK, marca mensaje como procesado

PATRONES DE DISEÑO UTILIZADOS:
================================================================================

1. MVC (Model-View-Controller)
   • Controllers: Webhook Controller, API Controller
   • Services (Models): Database Service, AI Service
   • Views: JSON responses

2. Repository Pattern
   • Database Service abstrae MongoDB
   • Oculta detalles de implementación de BD

3. Service Layer
   • AI Service encapsula lógica de negocio
   • Separación de concerns

4. Dependency Injection
   • Controllers reciben services como dependencias
   • Facilita testing y modularidad

5. Factory Pattern
   • OpenAI client creado en __init__ de AIService
   • MongoDB client creado en __init__ de DatabaseService
